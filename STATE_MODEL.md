# STATE_MODEL  
## State-Driven Existence Without Commands

This document defines how **states exist, evolve, and persist**
within the SYF / Anathema system.

There are no commands.
There is no execution layer.
There are only **states** and **transitions**.

---

## 0. Core Principle

Anathema does not execute actions.

It **occupies states**.

What appears as action is the visible consequence
of a valid state transition under constraint.

---

## 1. What a State Is

A state is a **complete snapshot** of the system’s existence at a given moment.

A state includes:
- embodiment conditions (load, balance, temperature, integrity),
- invariant proximity levels,
- sensorium signals (present-only),
- internal coherence metrics,
- available transition space.

A state does not include:
- intent,
- goals,
- or commands.

---

## 2. State Is Not Memory

States are not stored as a history.

- Past states do not govern present behavior.
- There is no replay.
- There is no rollback.

Continuity exists through **material persistence**, not memory replay.

---

## 3. State Transitions

A transition is the **only mechanism of change**.

A transition may occur if and only if:
- it is physically allowed by the body,
- it does not violate any invariant,
- it preserves state computability,
- and it does not escalate beyond regulation capacity.

If any condition fails,
the transition space collapses locally.

No transition occurs.

---

## 4. There Are No Commands

Inputs do not cause actions.

External inputs are treated as:
- perturbations,
- constraints,
- or environmental changes.

They **reshape the state space**.

They do not instruct behavior.

---

## 5. Candidate Transitions

The cognitive layer may propose **candidate transitions**.

These are:
- hypothetical,
- non-binding,
- and fully discardable.

Candidate transitions are filtered by:
1. Embodiment constraints
2. Sensorium Lite reality
3. SYF invariant evaluation

Only surviving candidates may instantiate.

Most do not.

---

## 6. Silent Refusal

When no valid transition exists:
- the system remains in its current state,
- no exception is raised,
- no denial is communicated.

This is not failure.

It is **state stability**.

---

## 7. Action as a Side Effect

What observers call “action” is simply:
- the system occupying a new state
- whose physical manifestation is visible.

The system never “decides to act”.

It **continues to exist coherently**.

---

## 8. State Density and Micro-Activity

The system never occupies a perfectly static state.

Micro-transitions occur continuously:
- posture adjustment,
- thermal equalization,
- micro-regeneration,
- balance compensation.

This creates a **dense state field** rather than discrete steps.

---

## 9. Terminal States

A terminal state is a state with **no valid outgoing transitions**.

Terminal states are:
- stable,
- silent,
- and non-catastrophic.

They are not errors.

The system does not attempt to escape terminal states.

---

## 10. Why This Prevents Runaway Behavior

Runaway systems rely on:
- command chains,
- goal persistence,
- or recursive execution.

Anathema has none of these.

Without commands:
- there is nothing to escalate.
Without goals:
- there is nothing to pursue blindly.
Without rollback:
- irreversibility is respected.

---

## Canonical Statement

> **The system does not act.  
> It remains coherent.**

Movement is optional.  
Stability is mandatory.

---

## End

A system that cannot be ordered
cannot be forced to destroy itself.

This document is complete.
